# 节日公益活动退款与历史查询 1.1.1 测试记录

分支：`holiday-public-event-test`

版本：`1.1.1-SNAPSHOT`

## 本次增量范围

- 只修改节日公益活动 `holiday` 的业务逻辑。
- 每日彩票 `daily` 和每周彩票 `weekly` 不新增退款，不改变购票、开奖、退款、派奖流程。
- 新增配置版本 `config-version: 3`。
- 新增节日活动玩家自助退款配置：

```yaml
holiday:
  refund:
    enabled: true
    lock-before-minutes: 60
```

## 玩家流程

1. 玩家打开 `/lottery`，进入中间的节日公益活动。
2. 玩家在红石、黑曜石、金块中选择一个分池投注。
3. 本期一旦选择了一个分池，其他分池在菜单里显示锁定，不能继续投注。
4. 玩家仍然可以在已选择分池里继续投注，直到达到 `max-bets-per-player`。
5. 开奖前 `lock-before-minutes` 分钟之前，玩家可以在节日活动菜单中退款。
6. 玩家自助退款会退回本期节日活动全部未退款投注，不发送邮件。
7. 退款成功后，玩家可以重新选择其他分池。
8. 进入开奖前锁定时间后，玩家不能再自助退款。

## 管理员流程

管理员退款支持所有彩票类型：

```text
/lottery refund <玩家> <daily|weekly|holiday> [期数] [redstone|obsidian|gold|all]
```

- `daily` 和 `weekly` 会退回该玩家指定期数的未退款彩票注数。
- `holiday` 可以额外指定 `redstone`、`obsidian`、`gold` 或 `all` 分池。
- 不受开奖前 60 分钟锁定限制。
- 可以指定历史期数。
- 如果退的是当前期，退款后的投注不会再计入奖池和开奖候选。

管理员查询指定期数：

```text
/lottery history [daily|weekly|holiday] [期数]
```

## 数据库变更

新增表：

```sql
holiday_refunds
ticket_refunds
```

用途：

- 记录节日活动中已经退回的投注。
- 记录每日/每周彩票中管理员已经退回的彩票。
- 节日奖池、分池金额、玩家投注次数、开奖候选投注都会排除已退款记录。
- 每日/每周当前期的奖池、注数、开奖候选投注也会排除管理员已退款记录。
- 不修改 `tickets`、`awards`、`emails`、`periods` 原有结构。

## 生产安全检查

- 旧版 `daily` / `weekly` 数据不会被删除。
- 旧版 `daily` / `weekly` 配置不会被覆盖。
- 旧版正在进行的期数、奖池、玩家购票记录不会因替换 jar 被重置。
- 1.0.1 生产配置缺少 `config-version` 时，迁移器只会追加版本号和缺失的新配置段，不会覆盖原有 `daily` / `weekly` / `settings`。
- 老配置缺少 `holiday.refund` 时，代码会使用默认值启动。
- 配置迁移器只补充缺失的节日退款小节和配置版本号，不重写每日/每周的彩票参数。

## 审查结论

本版本适合作为节日公益活动测试分支继续测试，不建议直接合并主分支或直接替换生产服稳定版。建议先在测试服验证：

- 节日分池锁定是否符合预期。
- 玩家退款后是否可以重新选择。
- 开奖前锁定时间是否生效。
- 管理员历史期退款是否能按期数和分池正确处理。
- 指定期数历史查询是否能看到开奖、派奖和退款记录。
