# 主分支稳定版记录

记录时间：2026-04-27

当前 GitHub `main` 分支保持为节日公益活动改动之前的稳定版本。

## 稳定版定位

该版本已经具备生产环境运行所需的核心能力：

- 每日彩票
- 每周彩票
- 箱子式菜单
- XConomy/Vault 经济对接
- SQLite 本地数据库
- SMTP 中奖邮件通知
- 配置热重载
- 手动开奖
- 奖池加注
- 高危重置
- 开奖广播
- PlaceholderAPI 基础变量

## 分支策略

- `main`：保留成熟生产版，不直接承载节日玩法实验。
- `holiday-public-event-test`：用于测试节日公益活动相关增量功能和后续动态奖池玩法。

## 升级原则

后续所有节日公益活动功能都应保持增量升级：

- 不覆盖生产环境已有 `config.yml`。
- 不删除或重置已有数据库。
- 不影响正在进行的每日彩票和每周彩票。
- 不改动已有奖池、期数和票据。
- 新配置只追加，不重写旧配置。

## 当前测试方向

节日玩法命名使用“节日公益活动”，外部表达保持健康合规。

下一阶段玩法方案将围绕建筑元素展开：

- 红石
- 黑曜石
- 金块

玩法核心应是玩家之间形成动态奖池，服务器只抽取公益手续费，不凭空生成货币。
